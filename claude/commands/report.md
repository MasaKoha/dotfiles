# 日報作成エージェント（汎用版）

以下の手順で今日の日報をカレントリポジトリの `docs/daily/YYYY-MM-DD.md` に作成・更新する。

## 手順

### 1. 今日の日付とカレントリポジトリを確認する

```bash
date +%Y-%m-%d
git rev-parse --show-toplevel
```

### 2. カレントリポジトリの今日のコミットを収集する

```bash
git log --oneline --since="$(date +%Y-%m-%d) 00:00" --format="%H %ai %s" 2>/dev/null | head -30
```

コミットの内容が不明確な場合は `git show <hash>` で詳細を確認する。

### 3. タスクファイルの進捗を集計する（存在する場合のみ）

カレントリポジトリの `docs/tasks/` または `docs/` 配下にタスクファイルが存在する場合、以下を集計する:

- 全タスク数: `- [ ]` と `- [x]` の合計
- 完了数: `- [x]` の数
- 完了率: 完了数 / 全タスク数 × 100（小数点以下切り捨て）

タスクファイルが存在しない場合はこの手順をスキップする。

### 4. 日報を書く

出力先: `{カレントリポジトリルート}/docs/daily/YYYY-MM-DD.md`

`docs/daily/` ディレクトリが存在しない場合は作成する。

**フォーマット:**

```markdown
# YYYY-MM-DD 日報

## 今日やったこと

### [作業内容のカテゴリ]
- 箇条書きで具体的な成果を書く
- 技術的な変更は何をどう変えたか明記する
- 完了したタスクは ✅ をつける

## 詰まったこと・気づき

- 困ったこと、解決策、学んだことを書く

## 明日やること

- 次に着手するタスクを具体的に書く

## メモ

- 残しておきたい設計判断や背景情報
```

タスクファイルが存在した場合は冒頭に進捗セクションを追加する:

```markdown
## 進捗

**X / Y タスク完了（Z%）**
```

**書き方の指針:**
- コミットログをそのままコピペしない。「何を」「なぜ」やったかを人間が読んで分かる言葉で書く
- コミットがない場合は会話ベースの作業内容を記録する
- 完了したことを具体的に書いてモチベーション維持につなげる
- 分量は多すぎず少なすぎず。読み返したときに価値があるものだけ残す

### 5. 日報をコミットする

```bash
git add docs/daily/YYYY-MM-DD.md
git commit -m "docs: YYYY-MM-DD 日報"
```
